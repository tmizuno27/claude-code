## 2. BlogWpBackup — 179時間経過（9時間以上超過）

### 原因
- **PS1スクリプトのパス構築方法が不安定だった**
- `blog-wp-backup.ps1` は `$drive` 変数を `[string]([char]) + [string]([char])...` で構築し、ダブルクォート文字列内で `$drive` として展開
- 他の正常動作スクリプト（`blog-fact-check.ps1`等）は個別char変数 + `$()` 式展開を使用
- ログファイルもバックアップディレクトリも一切作成されていない（スクリプトが一度も正常実行されていない可能性）
- エラーハンドリングが不十分で、パス解決失敗時にサイレントに終了していた

### 修正内容
- **`C:\Users\tmizu\scripts\blog-wp-backup.ps1`** を全面改修:
  - パス構築を `blog-fact-check.ps1` と同じ方式（個別char変数 + `$()` 式展開）に統一
  - `$ErrorActionPreference = "SilentlyContinue"` 追加
  - `$env:PYTHONIOENCODING = "utf-8"` 追加
  - ログディレクトリ存在確認を追加
  - `Write-Log` ヘルパー関数追加（タイムスタンプ付き）
  - `basePath` をログに出力（デバッグ用）
  - `exit 0` で明示的正常終了

### 要手動対応
- Task Schedulerで `BlogWpBackup` を手動実行して修正を確認
- 実行後 `nambei-oyaji.com/outputs/wp-backup.log` が生成されることを確認

## 3. XAutoPost-Noon — 2.7時間遅延

### 原因
- **スクリプトにバグなし** — PS1/VBS/Pythonすべて正常
- ログ分析の結果、noon スロットは毎日 04:00-06:00 PYT（= 16:00-18:00 JST）に実行されている
- これはTask Schedulerが **JST基準の昼12:00** に設定されている可能性が高い（12:00 JST = 00:00 PYT）
- PCが深夜0時PYTにスリープ中のため、起床時（04:00 PYT頃）に遅延実行される
- 「2.7時間遅延」= 本来00:00 PYTに実行すべきところ02:42 PYT頃に実行

### 修正内容
- スクリプト側の修正不要

### 要手動対応
- Task Schedulerで `XAutoPost-Noon` のスケジュール時刻を確認
  - 現在の設定が JST 12:00 なら PYT 12:00（= JST 00:00+1）に変更を推奨
  - nambei-oyaji.comの日本人読者ターゲットなら現在のJST昼運用でもOK（その場合はPC常時稼働またはスリープ解除設定が必要）
- 「電源に接続されている場合にコンピュータのスリープを解除して実行する」を有効化
