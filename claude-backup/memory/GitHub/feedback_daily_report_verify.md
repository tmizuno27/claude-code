---
name: 日次レポート更新時の情報鮮度チェック
description: 日次レポート更新時は全事業の情報が最新かを必ず検証してから更新する
type: feedback
---

日次レポートを更新する際は、全事業・全プロダクトのステータスが最新の実態と一致しているか必ずチェックすること。

**Why:** pSEOサイトがデプロイ済みなのにダッシュボードでは「待機中」のまま放置されていた。ハードコード値や古い情報が残り続けると判断を誤る原因になる。

**How to apply:**
- 日次レポート生成・更新の際、各事業のステータス（稼働中/待機中/停止中/審査中等）がメモリ・ダッシュボード・スクリプト間で一致しているか照合する
- 不一致があればレポート更新と同時にダッシュボード・メモリも修正する
- 特にハードコードされたステータス値は実態と乖離しやすいので重点チェック
