---
name: 自動化ファースト厳守
description: 手動作業を提案する前に必ず自動化手段を検討・構築すること。手動をデフォルトにしない
type: feedback
---

手動作業をユーザーに依頼する前に、必ず自動化できる方法を最優先で検討・構築・提案すること。

**Why:** ユーザーは「なぜ手動をやらせるのか。自動でできるならそうしろ」と明確に指摘。全事業・全タスク共通の厳守ルール。

**How to apply:**
- 外部サービスの操作が必要な場合、まずAPI/CLI/スクリプトで自動化できないか調べる
- APIトークンが必要な場合も、取得手順を最小化し、取得後は全て自動化する
- 「手動で○○してください」は最終手段。まず自動化を試み、不可能な場合のみ手動を提案
- 手動が必要な場合は「自動化できない理由」を明示すること
