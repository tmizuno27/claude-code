name: 全作業で収益最大化を最優先
description: 全ての作業・設計・スケジュール・意思決定において「収益最大化から逆算」を必須とする
type: feedback
---

全ての作業において、収益最大化を最上位の判断基準とすること。

**Why:** ユーザーから「とりあえず動くもの」ではなく「収益に最適化されたもの」を求められた。はてなブログのスケジュール設計で「とりあえず週3本」と組んだことを指摘された。

**How to apply:**
- スクリプト・自動化の設計時：「この設計は収益最大化から逆算されているか？」を必ず自問
- スケジュール設計時：ターゲットユーザーのアクティブ時間帯・プラットフォームのアルゴリズムを考慮
- 記事・商品の優先順位：収益直結度で並べ替え（高単価アフィリ記事 > 集客記事 > ブランディング記事）
- 新機能・新事業の提案時：ROI（投下時間あたりの期待収益）で評価
- 「動けばOK」で終わらせない。「収益に最適化されているか」まで検証してから完了とする
