# マスタープロンプト（コピペ用コマンド集）

## 日常的に使うコマンド

以下のコマンドをClaude Codeにそのままコピペして使う。
日付部分だけ書き換えればOK。

---

## コマンド1：メモから記事＋ツイートを一括生成

最もよく使うコマンド。メモを書いたらこれを実行するだけ。

```
以下の手順で今日のコンテンツを生成してください。

1. inputs/2026-02-28.md を読み込む
2. agents/writer-agent.md のルールに従い、note記事の下書きを作成する
3. agents/sns-agent.md のルールに従い、記事テーマに関連するX投稿案を3日分（9投稿）作成する
4. すべてを outputs/2026-02-28/ フォルダに出力する

出力ファイル：
- outputs/2026-02-28/article-draft.md（note記事の下書き）
- outputs/2026-02-28/tweets.md（X投稿案）
```

---

## コマンド2：1週間分のツイートをまとめて生成

毎週土曜日に実行。来週分のツイートを一括作成。

```
agents/sns-agent.md のルールに従って、
来週1週間分（月〜日）のX投稿案を作成してください。

テーマ：「（ここにテーマを入れる）」
1日3投稿 × 7日 = 21投稿

投稿タイミング（日本時間）も指定してください：
- 朝 8:00
- 昼 12:00
- 夜 20:00

outputs/tweets-week-YYYY-MM-DD.md に出力してください。
```

---

## コマンド3：ココナラ出品文を作成

新しいサービスをココナラに出品するとき。

```
agents/writer-agent.md のルールに従って、
以下のサービスのココナラ出品文を作成してください。

サービス内容：（ここに書く）
ターゲット：（誰向けか）
価格：（○○円）
納品期間：（○日）

出力：
- タイトル案 3つ（各25文字以内）
- 出品説明文（400文字以内）
- 購入時の注意事項 3つ
- よくある質問 3つ

outputs/coconala-listing.md に出力してください。
```

---

## コマンド4：競合・キーワード調査

記事を書く前に、どんなキーワードで書くべきか調査。

```
agents/research-agent.md のルールに従って、
以下のテーマでキーワード調査と競合分析を行ってください。

テーマ：「（ここにテーマを入れる）」

調査してほしいこと：
1. このテーマに関連する検索キーワード（noteとGoogle）
2. 既存の競合記事（note・ブログ）の分析
3. 差別化できるポイント
4. 記事タイトル案 5つ

outputs/research-YYYY-MM-DD.md に出力してください。
```

---

## コマンド5：スペイン語を翻訳

現地のニュースや文書を日本語にしたいとき。

```
agents/translator-agent.md のルールに従って、
以下のスペイン語テキストを日本語に翻訳してください。

---
（スペイン語テキストをここに貼り付け）
---

出力に含めること：
- 3行要約
- 全文翻訳
- この情報を記事にする場合のテーマ提案

outputs/translation-YYYY-MM-DD.md に出力してください。
```

---

## コマンド6：週次レビュー

毎週月曜日に実行。先週の振り返りと来週の計画。

```
agents/research-agent.md のルールに従って、
inputs/weekly-review-YYYY-MM-DD.md を元に週次レビューレポートを作成してください。

含めること：
1. 数値の前週比較と傾向分析
2. 良かった点・改善点
3. 来週の具体的アクション3つ（優先度付き）
4. 来週の記事テーマ提案 2つ

outputs/weekly-review-YYYY-MM-DD.md に出力してください。
```

---

## コマンド7：業務代行（ライティング案件）

ココナラやランサーズで受注した記事を作成。

```
agents/writer-agent.md のルールに従って、
以下のクライアント依頼内容で記事を作成してください。

【依頼内容】
テーマ：（テーマ）
キーワード：（SEOキーワード）
文字数：（○○文字）
ターゲット読者：（誰向けか）
トーン：（ですます調 / である調 / カジュアル）
参考URL：（あれば）
特別な指示：（あれば）

注意：
- この記事はクライアント納品用です（私の個人ブログではない）
- 一人称は使わず客観的に書く
- クライアントの指示を最優先する

outputs/client-article-YYYY-MM-DD.md に出力してください。
```
