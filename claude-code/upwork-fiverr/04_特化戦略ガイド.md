# 特化戦略ガイド

## 結論：特化一択

オールラウンダーではなく、特定分野に特化すべき理由と方法。

---

## オールラウンダーが負ける理由

クライアント目線で考える。

```
あなたが歯が痛いとき、どちらに行きますか？

A: 内科・外科・皮膚科・歯科・眼科なんでも診ます
B: 歯科専門 15年の実績

→ 全員Bを選ぶ
```

Upwork・Fiverr でも全く同じことが起きている。

---

## 数字で見る現実

```
「Japanese translator」で検索
→ 約2,000人がヒット。埋もれる。

「Japanese SaaS localization specialist」で検索
→ 20〜30人。選ばれる確率が100倍。
```

特化すると **競合が激減し、単価が上がる**。これが最大のメリット。

---

## 特化 vs オールラウンダー 比較

| | オールラウンダー | 特化型 |
|---|---|---|
| **競合数** | 数千人 | 数十人 |
| **単価** | 低い（価格競争に巻き込まれる） | 高い（専門家料金を取れる） |
| **提案の通りやすさ** | 低い（誰でも良く見える） | 高い（「この人しかいない」） |
| **リピート率** | 低い | 高い（専門家は替えが効かない） |
| **プロフィールの印象** | 「器用貧乏」 | 「プロフェッショナル」 |

---

## 特化の「仕方」が重要

### 悪い特化

```
「翻訳専門です」
→ まだ広すぎる。翻訳者は何万人もいる。
```

### 良い特化

```
「日本進出したいEC事業者向けの商品ページローカライズ専門です」
→ ターゲット × サービス × 分野 の3つが絞られている
```

### 特化の公式

```
誰に     ×  何を      ×  どの分野で

例1: 海外EC事業者に × 商品ページ作成を × Amazon Japan特化で
例2: SaaS企業に    × UIローカライズを  × B2B アプリ特化で
例3: 海外企業に    × 市場調査を       × 日本の食品業界特化で
```

---

## よくある不安への回答

### 「特化したら案件が減りませんか？」

```
逆。

オールラウンダー: 100件に応募 → 2件受注（受注率2%）
特化型:          20件に応募 → 5件受注（受注率25%）

特化した方が結果的に稼げる。
```

### 「でも最初はスキルがないので…」

**最初の1〜2ヶ月だけは例外。**

```
Month 1〜2: やや広めに受けて実績を5〜10件作る
Month 3〜:  実績をもとに特化を宣言する
```

最初から完璧に特化する必要はない。実績を作りながら「どの分野の依頼が多かったか」「どれが楽しかったか」で絞っていけばOK。

---

## 実践ステップ

```
Step 1  まず2〜3カテゴリで始める（実績作り）
        ↓
Step 2  最初の10件で「どれが得意か・楽しいか」を見極める
        ↓
Step 3  1つに絞ってプロフィールを書き換える
        ↓
Step 4  専門家として単価を上げる
```
