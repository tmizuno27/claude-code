# RapidAPI 全24本 SEO改善案（2026-03-25）

## 概要

全24 APIの売上・サブスクライバーともに$0/0人の現状を打破するため、RapidAPI Provider Dashboard上でのリスティングSEOを全面改善する。READMEは既に強化済み（2026-03-23完了）だが、**RapidAPI Studio上のTitle / Short Description / Tags**が検索発見性のボトルネック。

## 改善方針

1. **タイトル**: 「Free」「Fast」「No Auth」等の検索トリガーワードを前方配置
2. **Short Description**: 70-80文字。ユースケース+差別化ポイント+「Free」を含める
3. **Long Description**: 既存のコピペファイル（`docs/rapidapi-studio-copypaste.md`）を拡張
4. **Tags**: 各API 8-10個。汎用タグ+固有タグの組み合わせ

## 共通タグ（全APIに付与）

```
free api, no auth required, cloudflare workers, rest api, fast api
```

## 更新手順

1. https://studio.rapidapi.com/ にログイン
2. 各APIを選択 → **Details** タブ
3. **API Name** にタイトルをペースト
4. **Short Description** にショート説明をペースト
5. **Long Description** は既存の `docs/rapidapi-studio-copypaste.md` を使用（未作成分は本ファイルのShort Descriptionを拡張して記入）
6. **Tags** フィールドにタグをカンマ区切りで入力
7. **Save** → 次のAPIへ

## 優先順位（高→低）

検索ボリュームと競合状況から、以下の順に更新を推奨:

| 優先度 | API | 理由 |

| 1 | #12 Social Video Download | 動画ダウンロード系は検索需要が極めて高い |
| 2 | #02 Email Validation | B2B需要が大きく、有料転換率が高い |
| 3 | #13 Crypto Data | 仮想通貨データは常時需要がある |
| 4 | #01 QR Code Generator | QRコード生成は汎用性が高い |
| 5 | #06 IP Geolocation | セキュリティ・アナリティクス用途で安定需要 |
| 6 | #10 Currency Exchange | Fintech・EC系で需要あり |
| 7 | #14 SEO Analyzer | SEO需要は高いが競合も多い |
| 8 | #11 AI Text | AI系は検索爆増中 |
| 9 | #18 AI Translation | 同上 |
| 10 | #19 Trends Aggregator | ユニーク性が高い（唯一のマルチソース） |
| 11-24 | 残り | 上記完了後に順次対応 |
